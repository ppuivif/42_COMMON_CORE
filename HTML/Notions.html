HTML

En terme de sémantique, pourquoi utiliser "ligne" pour HTML et non ligne de code ?
Bien qu'HTML soit un langage de markup (balisage) plutôt qu'un langage de programmation, on peut néanmoins considérer que chaque élément HTML dans le fichier représente une "ligne de code" car il définit une action ou une structure qui sera rendue dans le navigateur.
En d'autres termes, chaque balise HTML ou attribut peut être vu comme une "instruction".

CSS

Le CSS est un langage descriptif (ou de style), conçu pour le design et la mise en page.
Il n'a pas les caractéristiques d'un code :
- il n'a pas de logique complexe : pas de boucles (for, while), ni de conditions complexes (if, else), ni de fonctions ou de structures dynamiques,
- il ne permet pas d'écrire des algorithmes ou de résoudre des problémes complexes,
- il ne peut pas effectuer des opérations logiques, ni manipuler des données.
Le CSS est uniquement déclaratif car il décrit comment les éléments HTML douvent être affichés sans "calculer" ou "décider".

Une règle CSS est un ensemble de styles appliqué à un ou plusieurs éléments HTML.
Une déclaration CSS correspond à une propriété et une valeur (par exemple, color: red;).
Les propriétés CSS sont les éléments spécifiques que tu modifies (comme color, margin, etc.).

Exemple de règle :
h1 {,
color: red; cette ligne est une déclaration
font-size: 24px; cette ligne est une autre déclaration
}
